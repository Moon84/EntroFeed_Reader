# Article Analyzer

深度分析文章内容，提取关键信息和结构化知识。

## When to Use

- 用户请求分析特定文章
- 需要提取文章的论点和证据
- 希望了解文章的核心内容和价值
- 对文章进行批判性评估

## How to Use

### 步骤 1: 获取文章内容

使用 `get_entry_content` 工具获取文章的完整内容：

```
get_entry_content(entry_id="<entry_id>")
```

### 步骤 2: 分析文章结构

分析提取的内容，识别：
- **主题**: 文章讨论的主要话题
- **论点**: 作者的核心观点
- **证据**: 支持论点的数据、引用或例子
- **结论**: 作者的最终结论或建议

### 步骤3: 补充关联资料
使用 `search_entries` `web_search` 工具搜索相关主题的其他文章：

```
search_entries(query="<query>")
web_search(query="<query>")
```
- **对比**：对比文章的观点和证据

### 步骤 4: 提取关键信息

输出结构化的分析结果：

```json
{
  "title": "文章标题",
  "main_thesis": "核心论点",
  "keyFindings": ["关键发现1", "关键发现2"],
  "supportingEvidence": ["证据1", "证据2"],
  "conclusion": "结论",
  "related_articles": ["相关文章1", "相关文章2"],
  "gaps": "文章的局限性或未解决的问题",
  "reference": "与用户兴趣的相关性评估"
}
```

## 分析维度

### 1. 内容分析
- 主题分类（技术/医学/金融/科学等）
- 内容质量评估
- 信息密度

### 2. 观点提取
- 作者立场
- 主要论点
- 论证逻辑

### 3. 证据评估
- 数据来源可靠性
- 引用权威性
- 样本大小（如适用）

### 4. 实用性评估
- 对读者的价值
- 可操作建议
- 相关领域


## Tools

- `get_entry_content`: 获取文章完整内容
- `get_user_interests`: 获取用户兴趣以评估相关性
- `search_entries`: 搜索相关主题的其他文章

## Example

```
用户: 分析一下这篇关于AI医疗的文章

助手:
1. 调用 get_entry_content 获取内容
2. 分析文章结构
3. 提取:
   - 主题: AI在医学诊断中的应用
   - 主要观点: 深度学习可以提高诊断准确率
   - 关键发现: 在肺癌早期检测中准确率达95%
   - 证据: 基于10万张CT扫描的临床试验
4. 补充关联资料：
  - 对比: 其他早期CT扫描对肺癌的诊断率，检出率等的结果，用来对比效果的可靠性
5. 输出结构化分析结果
6. 评估文章质量和适用性
```

## Best Practices

1. **先获取内容再分析** - 不要基于标题或摘要下结论
2. **识别信息来源** - 注意作者背景和发表平台
3. **保持客观** - 既要提取作者观点，也要指出潜在偏见
4. **关联用户兴趣** - 评估文章与用户关注领域的相关性
5. **确保来源真实有效** - 验证数据来源和引用来源的可靠性
