# SKILL: 麦肯锡式技术赛道深度研判

**McKinsey-Style Technology Track Deep Analysis**

**版本**: v2.0 | **执行阶段**: 4 阶段 | **策略提示词**: 10 个

## 核心定位

融合麦肯锡经典行业研究方法论（**MECE 分类、问题树拆解、80/20 抓核心、证据链闭环**），针对**AI 医疗、AI 大模型应用、生物医药技术、智能硬件**等技术驱动型细分赛道定制优化。

**摒弃**：冗余学术溯源与纯商业泛谈
**专注**：快速拆解技术脉络、吃透落地逻辑、研判商业化前景
**适配场景**：技术调研、项目立项、投资尽调、职业深耕
**核心能力**：用咨询思维快速吃透"技术 + 行业"双重脉络

## 四阶段执行流程 (4-Phase Execution Framework)

### Phase 1: Intake & Framing (需求澄清与框架设定)

**目标**: 明确分析目标、用户背景、输出期望，设定分析边界

**执行步骤**:
1. **需求澄清** - 用以下问题明确输入：
   - 用户背景：`<user_data field="user_background">...</user_data>`
   - 分析目的：`<user_data field="analysis_purpose">...</user_data>`
   - 时间 horizon：`<user_data field="time_horizon">...</user_data>`
   - 角色定位：`<user_data field="stakeholder_role">...</user_data>`

2. **框架选择** - 根据目的选择提示词：
   | 目的 | 推荐提示词 |
   |------|-----------|
   | 快速了解赛道 | Prompt 1 (MECE 界定) + Prompt 6 (PEST+ 壁垒) |
   | 项目立项 | Prompt 2 (双脉络研判) + Prompt 8 (商业化路径) |
   | 投资尽调 | Prompt 7 (竞争格局) + Prompt 9 (风险评估) |
   | 长期跟进 | Prompt 5 (信源矩阵) + Prompt 10 (战略合成) |

3. **边界设定** - 明确：
   - 分析深度：入门速览 vs 深度研判
   - 时间范围：近 1-2 年技术动态
   - 地理范围：全球/中国/特定区域

**输出**: 分析范围文档 + 提示词选择 + 预期输出结构

---

### Phase 2: Plan & Parallel Execution (计划与并行执行)

**目标**: 制定分析计划，并行执行多个提示词获取多维度洞察

**执行步骤**:

1. **制定计划** - 根据目标拆解任务：
   ```
   任务拆解示例（技术调研场景）：
   ├─ 任务 A：MECE 技术赛道界定 (Prompt 1)
   ├─ 任务 B：技术 + 行业双脉络研判 (Prompt 2)
   ├─ 任务 C：数据源与 API 映射 (Prompt 4)
   └─ 任务 D：PEST+ 壁垒分析 (Prompt 6)
   ```

2. **并行执行** - 同时执行多个提示词：
   - 任务 A → 获取技术边界和分层框架
   - 任务 B → 获取生命周期和驱动因子
   - 任务 C → 获取数据源和集成方案
   - 任务 D → 获取外部环境和内部壁垒

3. **中间检查点** - 验证：
   - 证据是否充分
   - 数据是否权威
   - 框架是否完整

**输出**: 多维度分析草稿 + 证据初稿

---

### Phase 3: Collect & Synthesize (信息收集与综合)

**目标**: 收集证据链，综合各维度分析，形成初步研判

**执行步骤**:

1. **证据收集** - 使用 Prompt 3 框架：
   | 级别 | 内容 | 最低要求 |
   |------|------|---------|
   | 一级核心证据 | 监管文件/顶会论文/白皮书 | ≥3 份权威来源 |
   | 二级佐证资料 | 研报/评测/开源项目 | ≥5 份 |
   | 三级辅助参考 | 博客/讨论/竞品拆解 | 交叉验证 |

2. **信源建立** - 使用 Prompt 5 建立长期信源矩阵：
   - 监管层：部委/协会/标准组织
   - 技术层：顶会/开源/专利
   - 商业化层：龙头/融资/试点
   - 舆情层：媒体/KOL/社群

3. **综合分析** - 合成各维度：
   ```
   综合框架 = MECE 框架 (Prompt 1)
            + 生命周期判断 (Prompt 2)
            + 竞争格局 (Prompt 7)
            + 壁垒分析 (Prompt 6)
            + 风险评估 (Prompt 9)
   ```

**输出**: 综合分析草稿 + 证据链台账 + 信源矩阵

---

### Phase 4: Delivery & Integration (交付与架构集成)

**目标**: 输出最终研判报告，生成项目架构完善建议

**执行步骤**:

1. **战略合成** - 使用 Prompt 10：
   - 执行摘要（3 句话结论 +5 个关键数据）
   - 3 个战略选项及推荐
   - 90 天优先行动清单

2. **项目架构建议** - 基于 DeepResearch 架构：
   - Ontology 层：概念/关系/实体建议
   - Connector 层：数据源/API/刷新计划
   - Agent 层：子 Agent/技能/工作流
   - Task 层：任务拆解和优先级

3. **质量检验** - 使用自检清单：
   - MECE 检验：不重不漏
   - 证据检验：数据支撑
   - 落地检验：可执行性
   - 时效检验：近 1-2 年资料

**输出**: 最终研判报告 + 项目架构完善建议 + 任务清单

## 一、MECE 技术赛道界定：划清边界 + 锚定核心

### 1.1 技术赛道精准定义（杜绝边界模糊）

```
【核心技术内涵】
├── 底层技术栈：[列出核心技术组件]
├── 核心技术原理：[用一句话概括技术本质]
├── 关键技术指标：[量化指标及行业基准]
├── 技术定位：通用技术 vs 垂直应用 / 原生技术 vs 集成技术
└── 技术独特性：与替代技术、互补技术、上游技术的边界

【应用范畴拆分】（MECE 分层）
├── 核心应用：[当前主要落地场景，占市场 80%]
├── 边缘应用：[次要场景，占市场 15%]
└── 潜在应用：[未来可能场景，占市场 5%]

【关联技术区分】
├── 替代技术：[可能被什么技术替代]
├── 互补技术：[需要配合什么技术使用]
└── 不可替代性：[本赛道的核心壁垒]
```

### 1.2 技术 + 行业双脉络研判

```
【技术生命周期定位】
├── 当前阶段：实验室突破 / 试点落地 / 规模商业化 / 成熟迭代
├── 技术拐点：[关键突破时间点或里程碑]
├── 瓶颈卡点：[制约大规模应用的核心问题]
└── 迭代周期：[技术更新频率]

【双核心驱动因子】（80/20 法则）
├── 技术侧（20% 关键变量）
│   ├── 算法突破：[核心算法进展]
│   ├── 算力升级：[硬件/云能力]
│   └── 数据积累：[数据规模与质量]
└── 行业侧（20% 关键变量）
    ├── 政策合规：[监管态度与资质要求]
    ├── 临床需求：[真实痛点强度]
    ├── 成本下降：[规模化后的成本曲线]
    └── 资本导向：[投融资热度]

【核心价值命题】
├── 解决的痛点：[具体行业问题]
├── 效率提升：[量化提升比例]
├── 成本优化：[量化降低成本]
└── 商业化壁垒：[技术/合规/数据/渠道]
```

### 2.2 收集范围 + 数量标准

```
【时间范围】
├── 核心技术动态：近 1-2 年
└── 合规/商业化资料：近 3 年

【覆盖视角】
├── 技术底层：原理、算法、架构
├── 合规审批：资质、认证、伦理
├── 临床落地：试点、实测、反馈
└── 商业化：定价、渠道、竞争

【数量标准】
├── 入门速览：≥3 份权威技术白皮书 + ≥5 篇高被引论文 + 核心合规文件
└── 深度研判：≥10 份技术研报 + 核心专利清单 + 龙头技术方案 + 试点数据
```

### 2.3 麦肯锡筛选原则

```
【权威性优先】
官方合规文件 > 顶会顶刊 > 龙头白皮书 > 普通媒体
→ 杜绝软文、无数据技术吹稿

【相关性优先】
只保留支撑技术拆解、落地研判的资料
→ 剔除纯理论学术冗余内容

【可验证优先】
技术指标可溯源、落地案例可查证、合规资质可查询
→ 杜绝孤证结论
```

## 五、麦肯锡核心拆解方法 + 落地输出

### 5.1 技术赛道专属方法论

#### MECE 技术拆解法
```
第一层：底层技术
├── 算法基础
├── 硬件支撑
└── 数据基础设施

第二层：算法模型
├── 核心模型架构
├── 训练方法
└── 推理优化

第三层：工程实现
├── 系统集成
├── 性能优化
└── 质量控制

第四层：场景落地
├── 核心场景
├── 边缘场景
└── 潜在场景

第五层：商业化
├── 定价模式
├── 渠道策略
└── 竞争格局
```

#### PEST+ 技术壁垒分析
```
【外部环境】
├── Policy 政策：监管态度、支持政策、限制政策
├── Economic 经济：市场规模、增速、资本热度
├── Social 社会：用户认知、接受度、替代意愿
└── Technology 技术：发展阶段、迭代速度、颠覆风险

【内部壁垒】
├── 技术壁垒：算法/专利/人才
├── 合规壁垒：资质/认证/审批周期
├── 数据壁垒：数据规模/质量/获取成本
└── 渠道壁垒：客户资源/合作关系/品牌
```

#### 假设驱动法
```
【核心假设】
假设 1：技术能在 [时间] 内达到 [指标]
假设 2：成本能降至 [水平] 实现盈利
假设 3：合规能在 [时间] 内获批

【验证路径】
├── 找对标案例：[类似技术落地情况]
├── 找反证：[失败案例/障碍因素]
├── 数据交叉验证：[多方数据比对]
└── 专家验证：[从业者访谈]

【结论】
→ 假设成立/部分成立/推翻，附证据链
```

### 5.2 技术赛道避坑原则

```
【麦肯锡咨询禁忌】

❌ 不唯技术论
→ 脱离合规、成本、场景的技术无商业价值
→ 拒绝技术崇拜，回归商业本质

❌ 不沉迷实验室数据
→ 聚焦落地实测指标
→ 忽略理想化试验数据

❌ 不忽视合规卡点
→ 技术型赛道监管往往滞后于技术
→ 提前预判政策风险

❌ 不滞后迭代
→ 技术迭代快，紧盯底层突破与合规更新
→ 动态修正判断
```
